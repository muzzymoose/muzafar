Project 04

Bike Store Database System

Role Data Engineer
Duration 2 Months
Tools PostgreSQL, Python, pandas

Overview

Designed and implemented a normalized relational database for a bike retailer, migrating scattered CSV files into PostgreSQL. Built a reusable Python ETL framework that automates data import with full transaction control and integrity constraints.

The Challenge

The retailer stored operational data across multiple Excel files — leading to redundancy, inconsistencies, and painfully slow reporting. Finding a customer's order history meant searching through several spreadsheets manually.

Approach

  • Designed 3NF-normalized schema: Stores, Staff, Customers, Products, Orders, Brands, Categories
  • Established foreign key constraints and referential integrity
  • Built Python ETL pipeline with psycopg2 for automated CSV imports
  • Implemented batch processing using COPY command for performance

Key Outcomes

  • Complex queries now execute in under 100ms (previously hours of manual lookup)
  • Import process reduced from 2 hours manual work to 5 minutes automated
  • Eliminated duplicate records and data inconsistencies
  • Reusable framework can import any CSV structure with minimal config
